18Tell me about a time you failed or made a mistake. In a code test interview about finding the nearest date for a flight I used an ordered map. This was the correct approach and to complete the task all I need to do was use a function called "floor" that gets the nearest element (i.e. flight date less than or equal to a given key date). I had never used that function and was trying to think of a way to avoid a scan but we had run out of time. The developer running the test failed me as a consequence. In real life you would easily figure this out. In general when I test candidates I don't penalize them for not knowing one thing - I am interested in what they can do and how they think. This is the spirit of collaboration to me. It takes but a minute to find a suitable function if one exists - and about ten minutes to write your own version if one doesn't exist. Anyhow - lesson learned - make sure you know the standard functions available for all your typical data structures.

21Tell me about a time you disagreed with someone.This often occurs and I handle disagreements with an open mind - discussing to explore the possibilities. I have learned many things this way - including that my own biases are not always the best way forward. For eaxmple - we had to solve a speed issue in presenting a list of patient appointments. A design flaw was caused by using all backend microservices and APIs based on NoSQL for all entities and not having any easy SQL solution for handling relationships. The design was great for Service Oriented design, data updates and scalability - but not for any reporting or providing a list of appointments and diagnosis summary. I deferred to the tech director who wished to avoid creating any duplicate (denormalized) table. Instead I enhanced the design to cache results, preload data and keep these caches warm. In the end the trade off was quite a good solution and good lessons were learned. Conclusion: discuss and evaluate all views without bias.

The time taken was 20% of predicted - keeping this important customer happy.
