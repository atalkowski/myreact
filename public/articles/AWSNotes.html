1.1. What is CLOUD computing

Way back the original idea was tried and dropped over 7-8 years by Oracle.
  • Definition: A model for enabling UBIQUITOUS, CONVENIENT, ON-DEMAND network access to a SHARED POOL of configurable resources (e.g. NETWORKS, SERVERS, STORAGE, APPS, SERVICES) that can be rapdily PROVISIONED and RELEASED with minimum management effort or service provider interaction.
  • Buzzwords: Automation of processes Self service High availability - you are abstracting the data sources

The 5 characteristic of the cloud

  • ON-DEMAND - instant
  • BROAD NETWORK ACCESS - anywhere
  • RESOURCE POOLING - 100's of CPU's
  • RAPID ELASTICITY - 100's of servers for scaling out
  • MEASURED SERVICE - pay for usage PAYG and service levels - SLAs.

The 3 service models

By the way - it helps to log into the trial AWS site to see how powerful this is - literally PAYG.
Iaas - infrastructure as a service
  • You don't have to worry about hardware complexity
  • Hardware is all abstracted and automated
  • There is orchestration and automation of the deployment
  • All these give us the elasticity we need
PaaS - platform as a service
  • developers develop against a particular infrastructure setup
  • application can provision
  • automating done by the developer
SaaS - software as a service
  • E.g. SalesForce
  • consuming an application or set of services that you can build on top of

The 4 deployment models

  • PRIVATE - on premises deployment or in public - not sharing the hardware
  • HYBRID - able to extend your private to public where the public extension is in the cloud.
  • COMUNITY - vertical cloud - e.g. for GOV cloud - a sector sharing resources and using certificates etc e.g. Health cloud.
  • PUBLIC - most popular - you can use IaaS - pooling of resources ( such as servers, storage) with others
