Reactive Flux/Mono Operators

In what follow interpret FLUX as Mono or Flux. See flux docs
  • map(e -> f(e)): applies function f to transform each item emitted by a Mono or Flux
  • flatMap( e -> f(T e, Publisher(<R> r): mapper function f is expected to transform items of e of type T asynchronously into Publishers of type r. Then it flattens those publishers into a single Flux through merging. Not guaranteed that item order is preserved
  • flatMapSequential(..etc..) as above but maintains the order
  • collectList(): converts a Flux of items type T into a Mono<List<T>>
  • switchIfEmpty(FLUX alternative): converts an empty Mono of Flux into some MFlux alternative e.g. Mono.just("Was empty")
  • defaultIfEmpty
  • merge
  • concat
  • mergeSequential
  • zip
  • doOnRequest(LongConsumer consumer) Add behavior (side-effect) triggering a LongConsumer when this Flux receives any request.
  • doOnEach(Consumer<? super Signal<T>> signalConsumer) Add behavior (side-effects) triggered when the Flux emits an item, fails with an error or completes successfully.
  • doOnError

Example 1: map test

@Test
void testMapOperator() {
  Flux<String> mapExample = Flux.just("one", "two", "three")
    .map(string -> string.toUpperCase().concat(" EXAMPLE"))
    .log();
  StepVerifier.create(mapExample)
    .expectNext("ONE EXAMPLE","TWO EXAMPLE","THREE EXAMPLE")
    .expectComplete()
  .verify();
}
